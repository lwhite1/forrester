## The pattern you're trying to explain

A **reference mode** is a graph of the problematic behavior over time. It's the target your model must reproduce -- the pattern you're trying to explain.

## Common reference modes

- **Exponential growth** -- accelerating increase (population, compound interest)
- **Goal-seeking** -- gradual approach to equilibrium (thermostat, market price)
- **Oscillation** -- repeated cycles (inventory, business cycles)
- **S-shaped growth** -- acceleration then deceleration (technology adoption)
- **Overshoot-and-collapse** -- growth beyond capacity then decline (resource depletion)

## Sketch before you build

Draw the reference mode by hand before building the model. The sketch doesn't need precise data -- it captures the qualitative shape. Is the behavior growing or declining? Smooth or oscillating? Does it reach equilibrium or overshoot?

## Multiple reference modes

A model often has several key variables, each with its own reference mode. An urban model might track population (S-shaped growth), housing availability (oscillation), and commute time (exponential growth). Sketch reference modes for each key variable.

The reference mode guides the next stage -- the dynamic hypothesis must explain why this particular pattern occurs.
