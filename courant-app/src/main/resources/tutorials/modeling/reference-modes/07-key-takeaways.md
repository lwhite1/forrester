## What you learned

- **Five-stage process** -- problem articulation, dynamic hypothesis, formulation, testing, policy design
- **Problem articulation** -- define a specific dynamic question before building anything
- **Reference modes** -- sketch the behavior pattern you're trying to explain
- **Dynamic hypothesis** -- explain the pattern through feedback structure, not external events
- **Formulation and testing** -- iterate until the model passes structure and behavior tests
- **Policy design** -- explore interventions, watch for policy resistance, test robustness

## Key insight

The modeling process is iterative. Each stage may send you back to refine earlier work. A model that was never tested against a reference mode is just speculation. A model without a clear problem statement is a solution looking for a problem.
