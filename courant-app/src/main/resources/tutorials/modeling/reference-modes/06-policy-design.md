## Using the model for insight

**Policy design** uses the validated model to explore interventions. The question shifts from "why does this behavior occur?" to "what could change it?"

## Exploring policies

- **Parameter changes** -- what if we doubled production capacity? Halved the delay?
- **Structural changes** -- what if we added a new feedback loop, like an early warning system?
- **Compare alternatives** -- run multiple policies and compare their trajectories on the same chart

## Policy resistance

Watch for **policy resistance** -- cases where the system's feedback structure defeats the intended intervention. Examples:

- Building more highways to reduce congestion attracts more drivers, restoring congestion
- Subsidizing housing construction raises land prices, offsetting the subsidy
- Cracking down on drug supply raises prices, increasing dealer profits and attracting more suppliers

The model reveals these counterintuitive outcomes before you implement the policy in the real world.

## Robustness

A good policy works across a range of parameter assumptions, not just for one specific set of values. Use parameter sweeps to test whether the policy remains effective when uncertain parameters vary. Policies that only work under narrow assumptions are fragile.
