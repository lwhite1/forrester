## Models are never "right"

Models are simplifications of reality. They leave out details, approximate relationships, and compress time. They're never "right" -- but they can be **useful** or **misleading**.

Testing builds **confidence** that your model is useful for its intended purpose. Sterman (2000) outlines a battery of tests that systematically probe a model's structure and behavior:

- **Structure tests** -- is the model consistent with reality?
- **Extreme condition tests** -- does it behave plausibly at the edges?
- **Behavior reproduction** -- does it generate the expected patterns?
- **Sensitivity analysis** -- which parameters matter most?

No single test proves validity. But together they build a cumulative case -- like a trial with multiple lines of evidence. A model that survives many tests earns trust. One that fails any test needs more work before it can guide decisions.

This tutorial walks through each type of test and how to apply them.
