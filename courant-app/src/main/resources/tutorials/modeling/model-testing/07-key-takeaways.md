## What you learned

- **Structure first** -- check units, conservation laws, and causal relationships before running the simulation
- **Extreme conditions** -- push parameters to 0, very large, and negative values to reveal structural flaws
- **Behavior reproduction** -- compare model output to the reference mode, focusing on pattern rather than exact numbers
- **Sensitivity analysis** -- identify which parameters the model depends on and which are secondary
- **Cumulative confidence** -- no single test validates a model; apply many tests and document results

## Key insight

Models are tools for thinking, not crystal balls. Testing doesn't prove a model is "right" -- it builds evidence that the model is useful for its intended purpose. A well-tested model earns trust; an untested model earns nothing.
